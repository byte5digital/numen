It's zero-migration (works with existing content), AI-powered (tone-aware, not generic), and API-first (integrates with your headless workflow).

## 3. FIVE KEY MESSAGE BULLETS

1. **Your Persona translates, not just your text** — Numen uses your configured brand voice (Persona) to guide AI translation. If your brand is playful, translations stay playful. If you use domain-specific terminology, the AI learns and applies it across languages. Generic machine translation produces generic copy. Numen produces *your* copy, in any language.

2. **Translation matrix: See coverage at a glance** — Dashboard view shows which content exists in which locales. Gaps are obvious. Admins can spot untranslated content, filter by locale, and prioritize workflows. No guessing which articles need localization. Works with your existing content — zero migration required.

3. **Space-level locale configuration** — Add languages once at the space level. Set locale order for UI pickers, define fallback chains, mark your default language. Every content item in the space can be translated to any configured locale. Scales to 50+ languages without re-configuration.

4. **Intelligent 5-step fallback chain** — Content missing in `pt-BR`? Numen automatically falls back to `pt`, then your fallback locale, then the space default, then `en`. Users always see something meaningful instead of 404s. Graceful degradation without extra work.

5. **Headless i18n API with smart caching** — REST API respects Accept-Language headers and X-Locale overrides. Frontend specifies `Accept-Language: fr` and gets French content (or fallback chain) without extra logic. Built for static site generators, mobile apps, and SPAs. Translations are cacheable — CDN-friendly.

## 4. COMPETITIVE ANGLES

### vs **DeepL API / Google Translate**
- DeepL/Google: Generic tone, no brand voice awareness, separate system to manage
- **Numen**: Persona-aware translation baked into the CMS, one system for content + localization
- **Win**: Your translated content sounds like your brand, not a generic translation service. Fewer API calls, lower costs, no separate translation vendor.

### vs **Contentful Localization**
- Contentful: Localization requires enterprise plan ($500+/month), migrations from existing content, no AI guidance
- **Numen**: Built-in at any tier, zero migration, AI tone-matching via Persona
- **Win**: 1/10th the cost, stays in your system, AI handles tone consistency automatically

### vs **Strapi i18n Plugin**
- Strapi: Manual translation only, no AI assistance, fallback chains basic, no Persona-aware guidance
- **Numen**: AI-powered with brand voice, smart fallback, translation matrix UI, API design optimized for headless
- **Win**: Automation + consistency. Strapi teams translate manually; Numen teams use AI and iterate.

### vs **Manual Translation Workflows**
- Manual: Slow (weeks to translate a site), expensive (translators/$2k+), inconsistent (tone varies), doesn't scale
- **Numen**: Minutes to translate (AI + human review), in-system (no hand-offs), tone-consistent (Persona-guided), scales to any language
- **Win**: Speed, consistency, cost. 80% of translation work is AI; 20% is human review + tweaks.

### Numen's Unique Advantage: **Persona-Aware AI Translation**

No other headless CMS uses your configured brand voice to guide translation. This unlocks:
- **Consistency**: Same tone in every language
- **Speed**: AI does 80% in seconds; humans review + tweak
- **Brand preservation**: "Friendly" brand stays friendly in French, Japanese, Portuguese
- **Domain terminology**: If your Persona includes "customer success" instead of "support," that term flows through all translations
- **Team handoff**: AI generates first draft; content team reviews in translation matrix; no rework of structure/tone
